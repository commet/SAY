# PlayMCP 제출 정보 — SAY 2.0

## 등록값

- KC 서버 이름: `say-family-notice-v3` (2.0 신규 빌드 예정)
- MCP 식별자: `SAY`
- PlayMCP 이름: `SAY 가족 안내`
- Endpoint 경로: `/mcp`
- 인증: 인증 사용하지 않음
- 프로토콜: Streamable HTTP, stateless JSON response

## 한 줄 소개

병원·관공서·보험·택배·아파트 공지를 개인정보는 지우고 위험은 먼저 막으며, 가족이 함께 끝내는 임시 행동 케이스로 바꿉니다.

## 상세 소개

부모님이 받은 병원 문자, 관공서 신청 안내, 보험료 납부 문자나 낯선 택배 링크는 가족방에서 여러 번 전달되지만, 누가 무엇을 확인했고 다음에 무엇을 해야 하는지는 쉽게 놓칩니다.

사이(SAY)는 공지를 곧바로 저장하지 않습니다. 먼저 서버에서 주민등록번호·전화번호·주소·금융번호·이메일·생년월일·이름·URL 내부 식별정보 등을 마스킹하고, 확인된 사실과 빠진 정보, 설명 가능한 분류 신뢰도, 기관명과 링크 도메인의 일치 여부, 사기 위험 신호를 보여줍니다. 사용자가 이 미리보기를 확인하고 동의한 경우에만 최대 24시간짜리 임시 케이스를 만듭니다.

생성된 케이스에서는 위험도, 선행 조건, 우선순위와 기한을 바탕으로 가장 안전한 다음 행동 하나를 제안합니다. 기관 사칭이나 고위험 신호가 있으면 문자 속 링크를 누르는 행동을 막고 공식 앱·홈페이지·대표번호 확인을 먼저 요구합니다. 가족은 무작위 케이스 코드로 같은 상태를 이어 보고, 일반 역할 단위로 행동을 맡고 완료할 수 있습니다. 버전 검사를 통해 다른 가족의 최신 변경을 덮어쓰지 않습니다.

사이는 원문과 인용문을 저장하지 않고, 케이스를 즉시 삭제하거나 자동 만료시킬 수 있습니다. 사용자가 원할 때만 결과를 제한된 선택지로 한 번 기록하며, 원문·자유서술·케이스 코드 없는 반복 신호가 충분히 쌓였을 때 합성 평가 사례와 개선 실험 후보를 만듭니다. 피드백이 코드를 직접 바꾸지는 않으며 모든 변경은 회귀평가와 사람 승인을 거칩니다. 의료·법률·금융 판단이나 진위 단정은 하지 않습니다.

## 데모 시나리오

1. 이름·전화번호·병원 예약 정보가 섞인 문자를 보내 `inspect_notice`의 마스킹 미리보기를 확인합니다.
2. 저장 동의를 받은 뒤 `create_case`로 케이스를 만듭니다.
3. 기관명과 다른 도메인의 링크가 있는 공지에서 `get_next_action`이 공식 채널 확인을 최우선으로 선택하는지 봅니다.
4. `update_action`으로 확인 행동을 완료하고, 다음 가족 행동이 열리는 과정을 보여줍니다.
5. 사용자가 원할 경우 `record_outcome`으로 구조화된 결과를 한 번 기록합니다.
6. `make_family_message`로 가족방 메시지를 만들고 `delete_case`로 즉시 삭제합니다.
