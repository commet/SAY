# 사이(SAY) — 가족 안내 운영 에이전트

사이는 가족이 받은 병원·관공서·보험·택배·아파트 안내를 **안전하게 확인하고, 역할을 나누고, 끝까지 닫는 케이스**로 바꾸는 PlayMCP 서버입니다.

단순 요약기가 아닙니다. 원문을 저장하기 전에 서버가 개인정보를 제거하고, 기관명과 링크 도메인의 관계를 점검하며, 위험한 경우 공식 채널 확인을 다른 행동보다 먼저 강제합니다. 사용자가 마스킹 결과를 보고 명시적으로 동의한 뒤에만 최대 24시간짜리 임시 케이스가 생성됩니다.

![SAY 대표 이미지](say-playmcp.png)

## 핵심 흐름

```text
공지 원문
  → inspect_notice: 개인정보 제거 · 분류 · 사실/빈칸/위험 분석
  → 사용자에게 마스킹 미리보기와 위험 표시
  → 명시적 동의
  → create_case: 임시 케이스 생성
  → get_next_action: 가장 안전한 다음 행동 하나 선택
  → update_action: 가족 역할별 진행 · 충돌 방지
  → record_outcome: 사용자가 원할 때만 구조화된 결과 기록
  → 완료 후 delete_case 또는 자동 만료
  → 비식별 반복 신호 → 합성 회귀 사례 → 사람 승인 개선
```

### 왜 에이전트인가

- **개인정보 게이트**: 원문과 인용문을 저장하지 않고, 80비트 10분짜리 단일 사용 검사 토큰만 발급합니다.
- **설명 가능한 분류**: 가중 신호, 점수 차이, 신뢰도, 대안 범주를 반환하고 신뢰도가 낮으면 분류 확인을 선행 행동으로 만듭니다.
- **근거 기반 상태 머신**: 확인된 사실, 빠진 정보, 위험 신호, 선행 조건으로 케이스 상태와 다음 행동을 결정합니다.
- **안전 우선 계획**: 기관-도메인 불일치나 고위험 신호가 있으면 문자 링크를 열지 않고 공식 앱·홈페이지·대표번호 확인을 먼저 요구합니다.
- **가족 협업**: 엄마·아빠·보호자 같은 일반 역할만 저장하고, 버전 조건으로 동시 수정 덮어쓰기를 막습니다.
- **수명주기 완결**: 즉시 삭제와 최대 24시간 자동 만료를 지원하며, 전체 케이스를 열거하는 API가 없습니다.
- **안전한 자가 개선**: 자유서술 없이 반복되는 실패 유형만 집계하고, 평가 게이트와 사람 승인을 통과한 최소 변경만 배포합니다.

사이는 의료·법률·금융 판단을 하지 않으며 메시지가 진짜라고 단정하지 않습니다.

## MCP 도구

| 도구 | 역할 |
|---|---|
| `inspect_notice` | 필수 사전 단계. 서버 측 마스킹, 분류, 출처 신뢰도, 사실·빈칸·위험 분석 후 10분 토큰 발급 |
| `create_case` | 미리보기 확인과 명시적 동의 후 단일 사용 토큰으로 임시 케이스 생성 |
| `check_scam_signals` | 저장 없이 링크·기관 불일치·긴급 압박·개인정보/결제 요구 점검 |
| `get_case` | 비밀 케이스 코드로 현재 상태와 버전 조회 |
| `get_next_action` | 위험도·선행 조건·기한을 바탕으로 가장 안전한 다음 행동 하나 계산 |
| `update_action` | 행동 담당·보류·완료 상태 갱신, 선택적 버전 조건으로 충돌 방지 |
| `make_family_message` | 확인된 사실과 확인할 내용만 담은 가족용 메시지 생성 |
| `list_open_cases` | 사용자가 직접 제공한 코드들의 열린 행동만 조회 |
| `record_outcome` | 사용자가 자발적으로 알려준 결과를 원문·자유서술·케이스 코드 없는 비식별 개선 신호로 한 번만 기록 |
| `delete_case` | 케이스 즉시 삭제 |

모든 도구는 PlayMCP용 설명, 입력·출력 스키마, 텍스트 응답과 공통 `structuredContent`, 5종 `annotations`를 제공합니다. 삭제 도구만 `destructiveHint: true`입니다.

## 개인정보와 보안 경계

- 주민등록번호, 전화번호, 이메일, 금융번호, 주소, 생년월일, 여권·사업자·기기 식별자, UUID, 이름과 URL의 사용자정보·경로·쿼리·fragment를 서버에서 강제 마스킹하고 bidi/control 문자를 제거합니다.
- 검사 캐시는 마스킹된 텍스트만 최대 10분 보관하고 토큰 사용 즉시 제거합니다.
- 케이스는 생성 시 `expiresAt`이 고정되며 최대 24시간 뒤 삭제됩니다. 조회나 수정으로 만료가 연장되지 않습니다. 기본 로컬 실행은 메모리 저장이고, 컨테이너는 마스킹된 케이스만 `/data`에 원자적으로 직렬 저장합니다.
- `SAY-XXXX-XXXX-XXXX-XXXX` 코드는 약 80비트 무작위 bearer secret입니다. 신뢰하는 가족에게만 공유해야 합니다.
- 요청 본문은 64KB로 제한되고 IP별 메모리 기반 요청 제한, 재시도 헤더, 요청 ID와 보안 응답 헤더를 적용합니다.
- OAuth/Key 인증은 아직 구현하지 않았습니다. 따라서 PlayMCP 등록 시 **인증 사용하지 않음**이 프로토콜상 정직한 선택입니다. 장기 저장이나 민감 데이터 원본 보관에는 이 버전을 사용하면 안 됩니다.
- `CARD_STORE_PATH`에는 마스킹된 단기 케이스만 저장하며 로드 시 스키마와 만료를 검증합니다. 쓰기는 큐로 직렬화되고 종료 시 flush됩니다.
- `record_outcome`은 선택 사항이며 케이스당 한 번만 받습니다. 비식별 범주 카운터는 케이스와 연결되지 않으므로 케이스 삭제 후에도 남을 수 있습니다.
- 개선 신호는 사용자 입력으로 간주해 신뢰하지 않습니다. 표본 5건 미만은 후보로 만들지 않고 어떤 피드백도 서버 코드를 직접 바꾸지 못합니다.

자세한 설계는 [아키텍처 문서](docs/architecture.md), [위협 모델](docs/threat-model.md), [운영 런북](docs/operations.md), [자가 개선 루프](docs/self-improvement.md)를 참고하세요.

## 로컬 실행

요구사항: Node.js 22 이상

```bash
npm ci
npm run build
npm test
npm run eval
npm run improve
npm start
```

- MCP Endpoint: `http://localhost:8080/mcp`
- Health Check: `http://localhost:8080/health`
- 환경변수: `PORT`(기본 `8080`), `HOST`(기본 `0.0.0.0`), `CARD_TTL_HOURS`(1~24, 기본 24), `CARD_STORE_PATH`(선택), `IMPROVEMENT_STORE_PATH`(비식별 집계 파일, 선택), `IMPROVEMENT_EVENT_LOG`(기본 `false`, 운영 승인 시에만 `true`), `TRUST_PROXY_HOPS`(기본 미신뢰, 검증된 역방향 프록시를 정확히 1~3단계 거칠 때만 설정)

MCP Inspector:

```bash
npx @modelcontextprotocol/inspector http://localhost:8080/mcp
```

## 검증

`npm run eval`은 6개 범주 40개 합성 공지에 대해 전체·범주별 분류 정확도, 분류 신뢰도, 기대 필드 재현율, 위험 규칙 재현율과 정밀도, 개인정보 잔존과 인용문 저장 여부를 수치로 출력합니다. 릴리스 기준과 한계는 [평가 계약](docs/evaluation.md)에 명시했습니다.

`npm test`는 개인정보, 근거 규칙, 동의·토큰, 출처 불일치, 안전한 행동 순서, 선행 조건, 멱등성, 동시 수정 충돌, 삭제와 MCP SDK 기반 `initialize → tools/list → tools/call` 흐름을 검증합니다.

`npm run improve`는 현재 평가 결과와 비식별 집계를 읽어 반복 실패 군집과 다음 실험 후보를 출력합니다. 자동 코드 변경은 하지 않습니다. 전체 릴리스 게이트는 `npm run quality` 한 명령으로 실행합니다.

## PlayMCP in KC

```bash
docker build -t say-mcp .
docker run --rm -p 8080:8080 say-mcp
```

멀티스테이지 `Dockerfile`은 PlayMCP in KC Git 소스 빌드에 사용할 수 있습니다. 런타임은 비루트 `node` 사용자, 컨테이너 헬스체크, `/data`의 마스킹 케이스·비식별 집계 저장을 사용합니다. 서버는 stateless Streamable HTTP 방식으로 `/mcp`의 POST 요청에 JSON 응답을 반환합니다.
